Java基础

什么是 JDK？什么是 JRE？
JDK 是 Java 开发工具包，包含 JRE、编译器 javac、调试工具、打包工具等，给开发人员使用。JRE 是 Java 运行环境，包含 JVM 和 Java 核心类库，主要用于运行 Java 程序。简单说，开发用 JDK，运行用 JRE，而 JVM 是真正执行字节码的虚拟机。

Java 语言有哪些特点？
Java 主要有几个特点：一是面向对象，支持封装、继承、多态；二是跨平台，依赖 JVM 实现“一次编写，到处运行”；三是自动垃圾回收，减轻内存管理负担；四是类型安全和异常机制比较完善；五是生态成熟，尤其在企业级开发、后端服务、中间件方向非常强。

面向对象和面向过程的区别？
面向过程更关注“步骤”，适合简单流程型问题；面向对象更关注“对象和职责划分”，适合复杂系统建模。面向对象的优点是可复用、可扩展、可维护性更好，特别适合多人协作和大型项目。实际开发中并不是完全对立的，底层实现往往也会结合过程化思想。

面向对象的三大特征是什么？
封装、继承、多态。封装是把属性和行为放到类里，并隐藏实现细节；继承是子类复用父类能力；多态是同一个接口或父类引用，在不同对象上表现出不同实现。封装提升安全性，继承提升复用性，多态提升扩展性。

重载和重写的区别？
重载发生在同一个类中，方法名相同但参数列表不同，和返回值无关，本质是编译期多态。重写发生在父子类之间，方法名、参数列表相同，子类重新实现父类方法，是运行期多态。重写时访问权限不能更严格，抛出的异常范围也不能更大。

== 和 equals() 的区别？
== 比较的是地址，基本类型比较的是值；equals() 默认也是比较地址，但很多类比如 String、Integer 重写了它，用来比较内容。所以判断两个对象“值是否相等”通常用 equals()，判断是否是“同一个对象”用 ==。

String 是最基本的数据类型吗？
不是。Java 的基本数据类型只有 8 种：byte、short、int、long、float、double、char、boolean。String 是引用类型，是一个类，只是因为使用特别频繁，看起来像基础类型。

什么是自动装箱和拆箱？
自动装箱是基本类型自动转成包装类型，比如 int -> Integer；拆箱是包装类型自动转成基本类型，比如 Integer -> int。这个机制提高了开发便利性，但也要注意空指针问题，比如 Integer a = null; int b = a; 会触发拆箱并抛出 NullPointerException。

浮点数精度问题怎么解决？
不能直接用 float 或 double 做金额等高精度计算，因为二进制无法精确表示某些十进制小数。实际开发里通常使用 BigDecimal，而且建议通过字符串构造，比如 new BigDecimal("0.1")，避免精度损失。

short s1=1; s1=s1+1; 为什么报错，s1+=1; 为什么不报错？
s1 + 1 会触发整型提升，结果是 int，赋值给 short 会报错。s1 += 1 本质上包含了隐式强制转换，编译器会帮我们处理，所以不会报错。但从可读性和类型安全角度看，最好显式写清楚。

集合与泛型
11. ArrayList 和 LinkedList 的区别？
ArrayList 底层是动态数组，随机访问快，尾部插入效率高，但中间插入和删除可能需要移动元素。LinkedList 底层是双向链表，插入删除某个已知位置的元素更有优势，但随机访问慢。实际项目里 ArrayList 用得远多于 LinkedList，因为查询场景通常更多，而且 CPU 缓存友好。

HashMap 和 Hashtable 的区别？
HashMap 线程不安全，允许 null 键和 null 值；Hashtable 线程安全，但性能较低，不允许 null。现在基本不推荐用 Hashtable，并发场景一般用 ConcurrentHashMap。

List、Set、Map 三者区别？
List 元素有序、可重复；Set 元素不可重复；Map 存的是键值对，key 不可重复，value 可重复。选择时主要看业务语义，比如需要保序且可重复通常用 List，需要去重用 Set，需要按 key 查值用 Map。

HashMap 底层结构是什么？
JDK 1.8 里 HashMap 底层是“数组 + 链表 + 红黑树”。先根据 key 的 hash 定位到数组桶位，发生哈希冲突时先挂链表，链表长度超过阈值并且数组容量足够时转为红黑树，优化查询性能。这样设计兼顾了平均性能和极端冲突场景。

ConcurrentHashMap 为什么比 Hashtable 更适合并发？
因为 Hashtable 基本是整表加锁，并发度低。ConcurrentHashMap 在 JDK 1.8 里采用 CAS + synchronized 的方式，只锁冲突桶位，粒度更细，并发性能更好。它适合高并发读写场景，但也不是所有复合操作都天然线程安全，像“先判断再更新”仍可能需要额外控制。

什么是泛型？有什么好处？
泛型本质是参数化类型，能让代码在编译期检查类型安全，减少强制类型转换。比如 List<String> 明确表示这个集合里只能放字符串。它的核心价值是提升可读性、复用性和类型安全。

什么是类型擦除？
Java 泛型是编译期生效，编译后会擦除为原始类型，比如 List<String> 和 List<Integer> 运行时其实都是 List。这就是为什么运行时拿不到完整泛型信息，也解释了很多泛型限制，比如不能直接 new T()。

并发与多线程
18. Java 中实现多线程有几种方式？
常见有三种：继承 Thread、实现 Runnable、实现 Callable 配合 Future。实际项目中更推荐线程池方式，比如 ExecutorService 或 ThreadPoolExecutor，因为线程的创建和销毁成本高，线程池更利于资源管理和系统稳定性。

start() 和 run() 的区别？
run() 只是普通方法调用，不会启动新线程；start() 才会真正创建线程并由 JVM 调度执行 run()。面试里这题主要考察对线程启动机制的理解。

sleep() 和 wait() 的区别？
sleep() 是 Thread 的静态方法，调用后线程进入 TIMED_WAITING，但不会释放锁。wait() 是 Object 的方法，必须在同步代码块里调用，调用后会释放当前持有的对象锁，等待 notify() 或 notifyAll() 唤醒。一个偏线程调度，一个偏线程协作。

notify() 和 notifyAll() 的区别？
notify() 随机唤醒一个等待线程，notifyAll() 唤醒所有等待线程，再去竞争锁。notifyAll() 更安全，能避免部分线程永久等不到的问题，但上下文切换成本更高。实际开发中如果条件复杂，通常更推荐 notifyAll() 或直接用 Condition。

volatile 有什么作用？
volatile 主要保证可见性和一定程度的有序性，不能保证复合操作的原子性。比如一个线程修改了 volatile 变量，其他线程能立刻看到最新值。典型场景是状态标记、双重检查单例里的实例变量。

synchronized 和 ReentrantLock 的区别？
两者都能实现互斥同步。synchronized 是 JVM 层关键字，使用简单，自动释放锁；ReentrantLock 是 JDK 提供的显式锁，支持可中断、公平锁、超时获取锁、多个条件队列等，更灵活。简单场景用 synchronized，复杂并发控制更适合 ReentrantLock。

什么是线程安全？
线程安全指多个线程同时访问共享资源时，程序仍能得到正确结果，不出现脏数据、丢失更新、状态不一致等问题。实现手段通常包括加锁、CAS、线程隔离、不可变对象、并发容器等。

线程池的核心参数有哪些？
核心是 corePoolSize、maximumPoolSize、keepAliveTime、workQueue、threadFactory、RejectedExecutionHandler。面试时我一般会补充一句：线程池不是越大越好，关键看任务是 CPU 密集还是 IO 密集，以及队列和拒绝策略是否合理。

execute() 和 submit() 的区别？
execute() 用于提交 Runnable，没有返回值；submit() 可以提交 Runnable 或 Callable，返回 Future，可获取结果、取消任务、感知异常。实际需要拿执行结果或捕获异常时，我更常用 submit()。

JVM
27. JVM 内存区域有哪些？
一般分为程序计数器、虚拟机栈、本地方法栈、堆、方法区。线程私有的是程序计数器、虚拟机栈、本地方法栈；线程共享的是堆和方法区。对象主要分配在堆中，类元数据在方法区，方法调用和局部变量主要在栈中。

堆和栈的区别是什么？
栈主要存局部变量、方法调用信息，生命周期随方法结束而结束，访问速度快；堆主要存对象实例和数组，由 GC 管理，生命周期更灵活。栈解决的是方法执行问题，堆解决的是对象存储问题。

什么是 GC？为什么需要 GC？
GC 是垃圾回收机制，用来自动回收不再被引用的对象，避免内存泄漏和手动释放内存的复杂性。Java 通过 GC 提高了开发效率和安全性，但 GC 本身也会带来停顿和性能开销，所以需要合理调优。

如何判断对象可以被回收？
主流做法是可达性分析。JVM 会从 GC Roots 出发向下搜索，如果某个对象无法从 GC Roots 可达，就可以认为它是可回收的。比起引用计数，可达性分析更能解决循环引用问题。

强引用、软引用、弱引用、虚引用的区别？
强引用最常见，只要还有强引用，对象就不会被回收；软引用在内存不足时会被回收，适合缓存；弱引用只要发生 GC 就可能被回收；虚引用本身不影响对象生命周期，主要用于跟踪对象被回收的过程。

什么是内存泄漏？Java 会有吗？
会有。虽然 Java 有 GC，但如果对象已经没有业务意义，却仍被引用链持有，就会形成内存泄漏。常见场景包括集合未清理、监听器未注销、静态集合长期持有对象、ThreadLocal 使用不当等。

MySQL
33. InnoDB 和 MyISAM 的区别？
InnoDB 支持事务、行锁、外键、崩溃恢复，适合高并发 OLTP 场景；MyISAM 不支持事务和行锁，读性能较好但现在用得少。实际业务库基本优先选择 InnoDB。

索引的作用是什么？
索引本质上是帮助数据库快速定位数据的数据结构，最常见的是 B+Tree。它能显著提高查询性能，但会增加写入成本和存储空间，所以不是越多越好，要结合查询模式设计。

为什么 MySQL 常用 B+Tree 做索引？
因为 B+Tree 高度低、磁盘 IO 次数少，范围查询能力强，叶子节点有序且适合扫描。相比二叉树更稳定，相比哈希更适合排序和区间查询，所以特别适合数据库场景。

事务的四大特性是什么？
ACID。原子性表示事务要么全成功要么全失败；一致性表示事务前后数据满足业务约束；隔离性表示并发事务之间互不干扰；持久性表示提交后的结果不会丢失。

MySQL 事务隔离级别有哪些？默认是什么？
有读未提交、读已提交、可重复读、串行化。MySQL InnoDB 默认是可重复读。不同隔离级别解决的问题不同，隔离越高，并发通常越低。

SQL 优化你一般从哪些方面做？
我一般从四层看：第一是执行计划，先看有没有走索引；第二是 SQL 写法，避免 select *、函数操作索引列、隐式类型转换；第三是索引设计，关注联合索引、最左前缀、覆盖索引；第四是架构层，必要时分库分表、缓存、读写分离。

Redis
39. 什么是 Redis？
Redis 是一个基于内存的高性能 key-value 数据库，支持多种数据结构，比如 String、Hash、List、Set、ZSet，还支持持久化、发布订阅、Lua、事务和集群。常用于缓存、分布式锁、计数器、排行榜、会话存储等场景。

Redis 为什么快？
主要有几个原因：基于内存，避免磁盘 IO；单线程模型减少锁竞争；数据结构设计高效；网络模型和 IO 多路复用使其在高并发下仍有很高吞吐。补充一点，Redis 6 之后网络 IO 有多线程，但命令执行核心仍是单线程思路。

Redis 常见数据类型及使用场景？
String 适合缓存对象、计数器、分布式锁；Hash 适合存用户信息等结构化字段；List 适合消息队列、最新列表；Set 适合去重、共同关注；ZSet 适合排行榜和带权重排序场景。

Redis 持久化机制有哪些？
主要是 RDB 和 AOF。RDB 是快照，恢复快但可能丢最后一次快照后的数据；AOF 是追加日志，数据更安全，但文件更大、恢复更慢。生产上很多场景会同时开启，两者结合。

什么是缓存穿透、击穿、雪崩？
缓存穿透是查一个数据库里也不存在的数据，请求直接打到 DB；击穿是某个热点 key 过期瞬间大量请求直接打 DB；雪崩是大量 key 同时失效，导致数据库压力激增。解决通常分别是布隆过滤器/空值缓存、互斥锁/逻辑过期、过期时间打散/多级缓存/降级限流。

Jedis 和 Redisson 的区别？
Jedis 更轻量，接近 Redis 原生命令；Redisson 在此基础上做了更高级的分布式抽象，比如分布式锁、延迟队列、信号量、读写锁等。简单缓存操作我会优先 Jedis 或 Lettuce，复杂分布式协作更适合 Redisson。

Spring / Spring Boot
45. 什么是 Spring IOC？
IOC 就是控制反转，把对象的创建和依赖关系交给容器管理，而不是在业务代码里手动 new。这样可以降低耦合，提高可测试性和可维护性。依赖注入就是 IOC 最常见的实现方式。

什么是 AOP？
AOP 是面向切面编程，用来把日志、事务、权限、监控这种横切关注点从业务代码中抽离出来。这样能减少重复代码，让主流程更清晰。Spring AOP 底层主要通过动态代理实现。

BeanFactory 和 ApplicationContext 的区别？
BeanFactory 是更基础的 IOC 容器，按需加载 Bean；ApplicationContext 功能更完整，支持国际化、事件机制、自动装配、AOP 整合等，是平时开发里更常用的容器。

Spring Bean 的生命周期大致是什么？
一般是实例化、依赖注入、Aware 回调、初始化前置处理、初始化方法、初始化后置处理、正常使用、容器关闭时销毁。面试时能把 BeanPostProcessor 和初始化/销毁回调说出来，通常就比较加分。

Spring Boot 为什么流行？
因为它约定优于配置，简化了传统 Spring 项目的大量 XML 和繁琐整合；内嵌 Tomcat，启动部署方便；Starter 体系完善，自动配置能力强；同时又保留 Spring 生态的扩展性，特别适合微服务和快速搭建后台系统。

Spring Boot 自动配置原理是什么？
核心是 @SpringBootApplication，其中包含 @EnableAutoConfiguration。启动时会加载自动配置类，根据类路径、配置项、条件注解是否满足来决定是否装配 Bean。简单说，就是“你引入了什么依赖、配置了什么参数，Spring Boot 就帮你自动配好大部分基础能力”。

@Autowired 和 @Qualifier 的作用是什么？
@Autowired 用于按类型注入 Bean；如果同类型有多个实现，就可能冲突，这时配合 @Qualifier 指定具体 Bean 名称。实际开发里，为了更清晰和更利于测试，我更推荐构造器注入。

Spring Boot 配置文件加载顺序怎么理解？
总体原则是“越具体优先级越高”，比如命令行参数高于配置文件，外部配置高于内部配置，application-prod.yml 这种 profile 配置会覆盖通用配置。线上问题排查时，配置优先级理解得越清楚越重要。

MyBatis / Kafka / Linux
53. 什么是 MyBatis？它的优缺点是什么？
MyBatis 是半自动 ORM 框架，核心是 SQL 和 Java 对象映射。优点是 SQL 可控、学习成本低、适合复杂查询；缺点是需要手写 SQL，项目大了以后维护成本会升高。对于复杂业务系统，我认为它的可控性其实是很大的优势。

#{} 和 ${} 的区别是什么？
#{} 是预编译占位符，会使用 PreparedStatement，能防 SQL 注入；${} 是字符串直接拼接，常用于表名、排序字段等场景，但有注入风险。原则上能用 #{} 就不用 ${}。

MyBatis 一级缓存和二级缓存的区别？
一级缓存是 SqlSession 级别，默认开启，同一个会话中同样查询可能直接命中缓存；二级缓存是 Mapper 级别，多个 SqlSession 可共享，但在分布式和复杂更新场景下使用要非常谨慎。现在很多项目更倾向把缓存放到 Redis，而不是强依赖 MyBatis 二级缓存。

Kafka 为什么适合做消息队列？
Kafka 吞吐高、可持久化、可水平扩展、顺序性和消费者组机制成熟，非常适合日志采集、异步解耦、削峰填谷、流式处理。它的优势不是“功能最全”，而是在大数据量和高吞吐场景下表现特别稳定。

Kafka 如何保证消息不丢失？
生产端要合理配置 acks=all、重试和幂等；Broker 端要保证副本数和 ISR 正常；消费端要控制 offset 提交时机，最好在业务处理成功后再提交。严格来说，Kafka 更接近“至少一次”，最终要靠业务幂等补齐。

Linux 中如何查看进程、端口、日志？
查看进程常用 ps -ef | grep 进程名 或 top；查看端口常用 netstat -tunlp 或 ss -tunlp；查看日志常用 tail -f、cat、less、grep。线上排查我最常用的是 grep 配合 tail -f，快速定位关键报错。

如何查看磁盘和内存使用情况？
磁盘空间一般用 df -h 看分区总量和剩余量，用 du -sh 看目录占用；内存一般用 free -h，进程级内存可以配合 top 或 ps。这类命令是后端排查线上问题的基础。

线上系统 CPU 飙高你怎么排查？
我会先用 top 找高 CPU 进程，再用 top -Hp pid 找高 CPU 线程，然后把线程 ID 转成十六进制，去 jstack 里定位对应线程栈。接着分析是死循环、频繁 GC、锁竞争还是某个热点 SQL/接口引起，再对症处理。这个回答能体现你有真实排障经验。
